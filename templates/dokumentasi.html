📘 Dokumentasi WebApp SOC

👤 Panduan Pengguna

WebApp SOC membantu tim analyst memproses log keamanan dan membuat event secara otomatis atau manual.

🔹 Dashboard: Unggah file .txt / .xml untuk parsing otomatis.

🔹 Buat Event: Isi data manual atau gunakan Raw Mode dengan format TSV.

🔹 Copy Template: Klik tombol 📋 untuk menyalin template event siap kirim.

🔹 Shift & Instansi: Pastikan memilih sesuai jadwal & lembaga terkait.

⚙️ Dokumentasi Teknis

Bagian ini menjelaskan arsitektur dan fungsi utama dari aplikasi WebApp SOC.

  • app.py — Core Flask app, mengatur routing dan proses utama.
  • /templates/ — File HTML untuk tampilan web.
  • /data/events_magnitude_list.csv — Daftar tingkat severity event.
  • create_manual_event_preview() — Fungsi pembentuk template event manual.
  • parse_raw_line() — Parsing 1 baris input raw (TSV) ke dictionary event.
Contoh struktur direktori: 
📦 WebApp_SOC
 ┣ 📂 database          ← tempat penyimpanan data/event
 ┣ 📂 input             ← mungkin tempat upload file txt/log input
 ┣ 📂 outputs           ← hasil pemrosesan / laporan event
 ┣ 📂 templates         ← file HTML (Flask Jinja2 template)
 ┣ 📂 static            ← file statis (CSS, JS)
 ┣ 📂  __pycache__      ← cache Python
 ┣ 📜 app.py            ← file utama Flask (kemungkinan entry point)
 ┗ 📜 main_logic.py     ← berisi logika pemrosesan utama

🛡️ Mitigasi & Referensi Event

Referensi singkat mitigasi untuk event yang sering muncul pada sistem SOC.

Nmap Scripting Engine Detection

  • Kategori: Reconnaissance
  • Severity: Medium
  • Mitigasi:
    • Blokir IP source pada EDL / Firewall
    • Review aktivitas port scanning
    • Update IDS/IPS signatures

HTTP Directory Traversal Attempt

  • Kategori: Web Exploitation
  • Severity: High
  • Mitigasi:
    • Update web server patch
    • Gunakan web application firewall (WAF)
    • Audit permission direktori web
